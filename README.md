# Browser Automation Skill

浏览器自动化操作工具，帮助用户快速初始化浏览器自动化环境并进行各种网页自动化操作。

## 功能特性

- 🔧 **一键初始化**：自动完成 Chrome 远程调试开启和 openclaw 安装
- 💬 **自然语言操作**：无需编写代码，用日常语言描述即可执行自动化任务
- 📸 **网页截图**：自动打开网页并保存截图
- 📝 **表单自动填写**：智能识别表单元素并自动填写提交
- 🔍 **数据提取**：从网页中提取所需信息
- 🖱️ **页面操作**：点击、滚动、等待等各种页面交互

## 快速开始

### 1. 安装 Skill

将 `browser-automation.skill` 文件安装到你的技能库中。

或者将当前目录添加到技能库目录下中。

```
git clone https://github.com/chalecao/browser-automation.git
```

### 2. 初始化环境

首次使用时，告诉助手：
```
我想用浏览器自动化，帮我初始化一下环境
```

助手会引导你完成：
1. 开启 Chrome 远程调试
2. 检查/安装 openclaw@2026.3.13

### 3. 开始使用

初始化完成后，直接用自然语言描述你想做的事情：

**网页截图**
- "帮我打开 https://example.com 并截图保存"
- "访问这个网页，然后截个图"

**表单填写**
- "帮我打开登录页面，输入用户名 admin 和密码 123456，然后点击登录"
- "访问这个表单，自动填写姓名、邮箱和电话，然后提交"

**获取页面信息**
- "帮我获取这个网页的标题和主要内容"
- "提取这个页面的所有链接"
- "获取这个商品页面的价格和描述信息"

**页面操作**
- "帮我在当前页面点击那个蓝色的提交按钮"
- "滚动页面到底部，然后点击加载更多"
- "等待页面加载完成后，点击确认按钮"

## 使用示例

### 示例 1：抓取网页数据
```
用户：帮我抓取这个网站的所有文章标题和链接
助手：我来帮你抓取网页数据。首先我会打开目标网站，然后提取所有文章的标题和链接信息。
```

### 示例 2：自动化登录
```
用户：帮我登录这个网站，用户名是 test@example.com，密码是 123456
助手：我来帮你自动登录。我会打开登录页面，填写用户名和密码，然后点击登录按钮。
```

### 示例 3：定时刷新页面
```
用户：每隔5秒刷新一次页面，直到页面上出现"任务完成"这几个字
助手：我会设置一个循环，每隔5秒刷新页面，并检查页面内容，直到出现"任务完成"为止。
```

## 环境要求

- Google Chrome 浏览器
- Node.js 14+（用于安装 openclaw）
- 9222 端口未被占用（Chrome 远程调试默认端口）

## 手动开启 Chrome 远程调试

如果自动开启失败，可以手动开启：

### macOS
```bash
open -a "Google Chrome" --args --remote-debugging-port=9222
```

### Windows
```bash
chrome.exe --remote-debugging-port=9222
```

### Linux
```bash
google-chrome --remote-debugging-port=9222
```

## 常见问题

**Q: 无法连接到 Chrome？**
A: 确保 Chrome 已启动并开启了远程调试，检查端口 9222 是否正确

**Q: openclaw 安装失败？**
A: 检查 Node.js 版本（需要 v14+），尝试使用 sudo 或管理员权限安装

**Q: 页面操作超时？**
A: 可以告诉助手"增加超时时间到60秒"或"等待更长时间"

**Q: 可以操作已经打开的页面吗？**
A: 可以，告诉助手"连接到我当前打开的页面"或"操作现有的标签页"

## 安全提示

⚠️ 远程调试会暴露浏览器控制权，请在可信环境中使用，不要在公共网络或不安全的设备上开启。
