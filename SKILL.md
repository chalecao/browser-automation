# 浏览器自动化操作

## 概述

此 skill 帮助用户完成浏览器自动化操作的完整流程，包括：
1. 初始化环境（开启 Chrome 远程调试、检查/安装 openclaw）
2. 执行浏览器自动化任务

## 使用流程

### 第一步：检查初始化状态

首先检查用户是否已经初始化过：
- 询问用户是否已经完成了初始化
- 如果用户确认已初始化，跳转到「第四步：提供使用示例」
- 如果用户未初始化或不确定，继续执行初始化流程

### 第二步：开启 Chrome 远程调试

1. 指导用户打开 Chrome 浏览器的远程调试页面：
   - 让用户在 Chrome 地址栏输入：`chrome://inspect/#remote-debugging`
   - 或者使用 `open` 命令打开（macOS）：`open -a "Google Chrome" "chrome://inspect/#remote-debugging"`

2. 指导用户开启远程调试：
   - 找到 "Allow remote debugging for this browser instance" 选项
   - 勾选前面的复选框
   - 确认开启成功

3. 如果自动开启失败：
   - 告知用户手动开启的方法
   - 提示用户需要在 Chrome 启动参数中添加 `--remote-debugging-port=9222`
   - 等待用户确认已手动开启

### 第三步：检查并安装 openclaw

1. 检查当前 openclaw 版本：
   ```bash
   openclaw --version
   ```

2. 如果版本低于 2026.3.13：
   - 询问用户是否要更新到 2026.3.13 版本
   - 如果用户确认，执行安装：
     ```bash
     npm i -g openclaw@2026.3.13
     ```
   - 等待安装完成，验证版本

3. 如果未安装 openclaw：
   - 询问用户是否要安装 openclaw@2026.3.13
   - 如果用户确认，执行安装：
     ```bash
     npm i -g openclaw@2026.3.13
     ```

4. 安装完成后，验证安装成功：
   ```bash
   openclaw --version
   ```

### 第四步：提供使用示例

初始化完成后，给用户提供几个自然语言描述的操作示例：

**示例 1：打开网页并截图**
你可以这样告诉我：
- "帮我打开 https://example.com 并截图保存"
- "访问这个网页，然后截个图"
- "打开网站并保存页面截图"

**示例 2：自动化表单填写**
你可以这样告诉我：
- "帮我打开登录页面，输入用户名 admin 和密码 123456，然后点击登录"
- "访问这个表单，自动填写姓名、邮箱和电话，然后提交"
- "在这个页面自动填写所有必填项并提交表单"

**示例 3：获取页面信息**
你可以这样告诉我：
- "帮我获取这个网页的标题和主要内容"
- "提取这个页面的所有链接"
- "获取这个商品页面的价格和描述信息"

**示例 4：页面操作**
你可以这样告诉我：

- "滚动页面到底部，然后点击加载更多"

### 第五步：执行用户的具体任务

根据用户的自然语言描述，帮助用户完成浏览器自动化操作。用户只需要用日常语言描述想要做什么，比如：

- "帮我登录这个网站"
- "把搜索结果保存下来"
- "帮我填写这个表格并提交"
- "每隔5秒刷新一次页面，直到出现某个文字"

我会理解用户的意图并执行相应的自动化操作。

## 注意事项

1. **远程调试端口**：默认使用 9222 端口，如果该端口被占用，可以使用其他端口
2. **权限问题**：某些操作可能需要管理员权限
3. **浏览器版本**：确保 Chrome 浏览器版本与 openclaw 兼容
4. **安全性**：远程调试会暴露浏览器控制权，请在可信环境中使用

## 常见问题

**Q: 无法连接到 Chrome？**
A: 确保 Chrome 已启动并开启了远程调试，检查端口是否正确

**Q: openclaw 安装失败？**
A: 检查 Node.js 版本（需要 v14+），尝试使用 sudo 或管理员权限安装

**Q: 页面操作超时？**
A: 增加超时时间：`await page.goto('https://example.com', { timeout: 60000 })`
